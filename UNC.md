# Personal Knowledge Vault

Welcome to my second brain — a living collection of notes, ideas, reflections, and research.

All files in this repository follow a consistent **Universal Naming Convention (UNC)**. This system makes every note instantly searchable, sortable by topic, type, and maturity level. It turns a folder of Markdown files into an organized, evolving knowledge base.

## Universal Naming Convention (UNC)

Every file is named using this exact structure:  
**[AreaCode]-[TypeCode]-[StateCode]--[slug].md**

### Why this convention?
- **Quick orientation**: At a glance, you know *where* the note belongs, *what kind* of content it is, and *how mature* it is.
- **Searchable & sortable**: GitHub search, Obsidian, or any tool can filter by prefix (e.g., all `SEL-ID-*` files).
- **Lifecycle awareness**: Ideas grow from rough drafts to finished pieces without losing history.
- **Future-proof**: No more guessing what a file contains or whether it’s still in progress.

### Area Codes (Navigational markers — Where this lives)
These indicate the primary life or work domain:

- **FAI** — Faith / theology  
- **MAR** — Marriage  
- **SEL** — Self (cognition, identity, mental models)  
- **CHI** — Children / parenting  
- **FAM** — Extended family  
- **COM** — Community / society  
- **SYS** — Systems / operations / meta  
- **RES** — Research / technical  

*These are purely navigational — they carry no moral or priority weight.*

### Type Codes (Genre categories — What kind of content it is)
- **ID** — Idea / musing (exploratory thoughts, theories in formation)  
- **AR** — Article / essay (more developed writing)  
- **RS** — Research (formal notes with data, citations, or technical depth)  
- **WF** — Workflow / process / method (how-to guides, repeatable systems)  
- **PF** — Profile (about a person, concept, or myself)  
- **NT** — General notes  
- **IN** — Inbox / capture (raw, unprocessed items)

### State Codes (Lifecycle stage — How developed the note is)
- **DR** — Draft (early stage, incomplete)  
- **AIP** — Answers in progress (actively being worked on)  
- **PEN** — Pended / paused (on hold)  
- **FN** — Finished (complete but not necessarily published)  
- **LIV** — Live / published (ready for sharing or already public)  
- **ARC** — Archived / retired (no longer active)

This lifecycle approach treats every note like a living organism that can evolve over time.

## Quick Start & Usage Tips

- **Choosing codes**: Pick the most relevant Area first, then the best-fitting Type. Use **ID** for most exploratory or theoretical notes rather than forcing them into **RES**.
- **Slug**: A short, descriptive, lowercase name using hyphens (e.g., `mental-model-fragments`).
- **Example filename**: `SEL-ID-DR--mental-model-fragments.md`

**Pro tip**: Reserve the **RES** area for rigorous research with sources and data. Most thinking-out-loud pieces work best as **ID**.
